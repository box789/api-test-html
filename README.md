# 🧪 API Tester — инструмент для тестирования API прямо в браузере

> Postman-подобный инструмент, работающий полностью на стороне клиента (без бэкенда).

## 📌 Возможности

- 📋 Отправка HTTP-запросов с различными методами: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`
- 📝 Редактирование заголовков запроса (`headers`) и тела запроса (`body`)
- 📤 Отображение отправленного запроса и ответа сервера
- 📊 Просмотр статуса ответа, заголовков (`Response Headers`) и тела (`Response Body`)
- 🍪 Управление куками: сохранение, редактирование и отображение кук по доменам
- 🎨 Современный, тёмный UI с адаптивной версткой
- ⚙️ Автоматическое добавление заголовка `Content-Type: application/json` (если не задан явно)

## 💡 Технические особенности

- **Встроенный CORS-совместимый запрос** через `fetch()` — работает с серверами, включая кросс-доменные (с предварительным `OPTIONS`-запросом, если требуется)
- **Сохранение кук по доменам** с возможностью редактирования и повторного использования
- **Разделение вкладок ответа** (Request / Status / Headers / Cookies / Body) — для удобства анализа
- **Подсветка JSON** (через стандартный `<pre>` и цветовую разметку в HTML) — базовая, без внешних библиотек
- **Анимация статуса отправки** — индикатор «Sending...»

## 🛠️ Использование

1. Откройте `index.html` в любом современном браузере.
2. Введите URL API-эндпоинта.
3. Выберите метод и, при необходимости, добавьте заголовки и тело запроса.
4. Нажмите **Send** — результат появится в панели ответа.

> 💡 **Примечание:** Кросдоменные запросы требуют поддержки CORS на сервере. Внутри одного домена (same-origin) preflight-запросы не отправляются.
